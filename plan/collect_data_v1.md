# 大众点评 YOLO 数据采集 Plan v1

## 目标

为大众点评 APP 的 YOLO 目标检测模型采集训练数据。
当前阶段：**Phase 0 — 最小验证集**，覆盖 1-2 个核心页面，验证 VL 标注质量。

---

## 架构

```
手机端 (AutoJs6)          服务器 (Python)
┌─────────────┐           ┌──────────────────────┐
│ 自动浏览页面  │  base64   │ api_server.py         │
│ 截图+上传    │ ────────→ │  ├─ 保存截图           │
│ (快速,2s/张) │   JSON    │  ├─ 记录SQLite(防重复) │
│             │           │  └─ 返回成功(不等VL)    │
└─────────────┘           │                       │
                          │ 异步 VL 标注线程        │
                          │  ├─ Qwen2.5-VL-72B    │
                          │  ├─ 输出YOLO格式bbox   │
                          │  └─ 写入SQLite         │
                          └──────────────────────┘
```

**关键设计决策：**
- 采集和 VL 标注完全异步，采集不等待标注
- MD5 去重，相同截图不重复采集
- SQLite 记录每张截图的状态：pending → annotating → done/error

## Phase 0：首页 + 商家详情页

### 为什么选这两个页面

| 页面 | 理由 | 核心元素 |
|------|------|----------|
| 首页 | 最高频，所有操作起点 | 搜索栏、内容卡片、底部导航、发布按钮 |
| 商家详情 | 核心转化页面 | 店名、地址、人均、评分、笔记列表 |

### 采集流程

**首页（~4张）：**
1. 打开大众点评，等待加载
2. 截图：首页-初始
3. 上滑 × 3，每次截图：首页-滑动1/2/3

**商家详情（~9张，3个商家）：**
1. 回到首页顶部
2. 找到可点击的内容卡片 → 点击进入
3. 验证是否进入了详情页（检测"地址"/"人均"/"营业"文字）
4. 截图：商家详情-顶部
5. 向下滑动 × 2，每次截图
6. 返回首页，滑到下一个卡片
7. 重复 × 3 个商家

**预估：一轮 ~13 张不重复截图**

### 防重复机制
- 每张截图计算 MD5
- 相同 hash 的截图跳过（不保存、不标注）
- 采集不同时间段/不同滚动位置自然产生不同截图

## Phase 0 验收标准

1. 一轮采集跑完，无崩溃
2. 采集统计 API 返回 ≥ 10 张有效截图
3. VL 标注成功率 ≥ 80%
4. 抽查 3 张标注结果，bbox 位置基本准确（误差 < 15%）
5. 首页和商家详情页的 page_type 识别正确

**验收不通过 → 分析原因 → 修复 → 重跑，不进入 Phase 1**

      "x_center": 0.5,  // 归一化 0-1

}
